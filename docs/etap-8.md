# Этап 8: история проверок и базовое логирование

Документ фиксирует, **что реально сделано** на этом этапе в репозитории Range Program. Цель этапа: **накапливать** результаты каждой успешной проверки (`check`) в отдельном JSON-файле, дать команду **`range history`** для просмотра последних записей, добавить **файловое логирование** важных действий и ошибок, слегка обогатить **`show`** — **без** backtest, **без** графиков и **без** сложной аналитики.

Связанные документы: [`plan.md`](plan.md); этап 7 (массовый `check`) — [`etap-7.md`](etap-7.md); этап 9 (backtest диапазона) — [`etap-9.md`](etap-9.md).

---

## Цель этапа

1. **Не заменять** поле **`last_check`** на монете историей, а **дополнять** его **отдельным журналом** всех сохранённых проверок.
2. Хранить историю в **`data/check_history.json`** списком записей с ключевыми полями из **`CheckResult`** (цена, диапазоны, статус, рекомендация, проценты, время).
3. Вынести работу с файлом в **`CheckHistoryRepository`** (чтение/запись JSON, устойчивость к отсутствию файла, пустому содержимому и повреждённому JSON).
4. После каждой успешной **`evaluate`** в **`run_check`** (включая каждую монету в **`check --all`**) вызывать **`save_check(result)`**.
5. Добавить **`range history`**, опционально **`--all`** и **`--limit`**, плюс лог **`data/logs/app.log`** через стандартный **`logging`**.

---

## Что сделано

### 1. Файл истории и формат записи

- Путь: **`data/check_history.json`** (каталог создаётся при первой записи).
- Содержимое: **JSON-массив** объектов; новая проверка **дописывается в конец** (накопление, не перезапись всего «снимка монеты»).
- В каждой записи сохраняются поля, достаточные для последующего анализа и для вывода таблицы: **`symbol`**, **`current_price`**, **`active_low`**, **`active_high`**, **`recommended_low`**, **`recommended_high`**, **`status`**, **`recommendation`**, **`deviation_from_center_pct`** (в JSON — от **`deviation_from_active_center_pct`** в **`CheckResult`**), **`distance_to_lower_pct`**, **`distance_to_upper_pct`**, **`checked_at`** (ISO 8601).

Репозиторий: **`src/range_program/repositories/check_history_repository.py`**.

- **`save_check(result)`** — загрузить список, **`append`**, сохранить с отступами (**`indent=2`**).
- **`get_history(symbol)`** — все записи по символу (порядок как в файле).
- **`get_last_n(symbol, n)`** — последние **n** записей по монете, **от новых к старым** (сортировка по **`checked_at`**).
- **`get_global_last_n(n)`** — последние **n** записей **по всем символам**, по времени **`checked_at`**.
- **`get_all()`** — полный список как в файле.
- Если файл **отсутствует** или **пустой** — работаем с **пустым списком**; если JSON **не парсится** или не массив — в лог ошибка, список считается **пустым** (дальнейший **`save_check`** может перезаписать файл валидным массивом).

Экспорт из пакета: **`range_program.repositories.CheckHistoryRepository`** (см. **`repositories/__init__.py`**).

---

### 2. Интеграция с `CheckService`

Файл: **`src/range_program/services/check_service.py`**.

- В конструктор добавлен необязательный аргумент **`history: CheckHistoryRepository | None`** (по умолчанию **`None`**, чтобы тесты и скрипты могли не подключать журнал).
- После успешного **`update_coin`** с новым **`last_check`** вызывается **`self._history.save_check(result)`**, если репозиторий передан.
- При ошибках **рынка** (**`MarketDataError`**) и **оценщика** (**`EvaluatorError`**) перед пробросом как **`ValidationError`** пишется **`warning`** в логгер **`range_program.check_service`** (ошибки API/вычислений фиксируются в файле, если настроен root/иерархия логгеров).

**`last_check`** на **`Coin`** по-прежнему хранит **только последнюю** проверку; история — отдельно.

### 3. Вывод истории в терминал

Файл: **`src/range_program/history_view.py`**.

- **`print_history_entries(entries)`** — таблица колонок **DATE**, **PRICE**, **STATUS**, **DEV**, **DIST↓**, **DIST↑**; при пустом списке — сообщение **«(нет записей в истории)»**.

---

### 4. Логирование

Файл: **`src/range_program/logging_config.py`**.

- **`setup_logging()`** — один раз на процесс: **`FileHandler`** на **`data/logs/app.log`**, уровень **INFO**, формат с временем, уровнем и именем логгера; каталог **`data/logs/`** создаётся при необходимости.
- **`get_logger(name)`** — дочерние логгеры вида **`range_program.<name>`**.
- Ротация и лимит размера **не** реализованы; код изолирован так, чтобы позже добавить **`RotatingFileHandler`** или обрезку истории без переписывания CLI.

В **`cli.py`** при импорте вызывается **`setup_logging()`**; логгер **`range_program.cli`** используется для:

- глобального **`@app.callback()`**: подкоманда и строка аргументов (**`invoke … argv=…`**);
- **`add`**, **`remove`**, **`recalc`** (успех / предупреждение при сбое), **`check`** (одна монета и **`--all`**), **`history`**, **`set-active`**, **`clear-active`**.

В **`src/range_program/services/market_data.py`** при ошибках **`fetch_ticker`** / **`fetch_ohlcv`** пишется **`warning`** в **`range_program.market_data`**.

---

### 5. CLI: команда `history` и доработка `show`

Файл: **`src/range_program/cli.py`**.

- Создаётся **`CheckHistoryRepository()`** и передаётся в **`CheckService(..., history=_history_repo)`**.

| Вызов | Поведение |
|--------|-----------|
| **`range history BTC`** | Последние **10** записей по **BTC** (таблица). |
| **`range history BTC --limit 20`** (или **`-n 20`**) | Последние **20** записей по монете. |
| **`range history --all`** | Последние **20** записей **по всем** символам (смешанный хронологический срез). |
| **`range history --all --limit 50`** | То же с лимитом **50**. |
| **SYMBOL** и **`--all`** вместе | Сообщение об ошибке, код **1**. |
| Ни символа, ни **`--all`** | Сообщение об ошибке, код **1**. |

Команда **`show`**:

- Дополнительно выводятся **`last_status`** и **`last_checked_at`** (если **`last_check`** есть); блок **`last_check`** сокращён до **рекомендации** без повторения статуса и времени (они уже вынесены выше).

---

### 6. Тесты

Файл: **`tests/test_check_history_repository.py`**.

Проверяются: отсутствующий файл, пустой файл, повреждённый JSON, два **`save_check`** подряд, **`get_last_n`** (порядок от новых к старым), **`get_global_last_n`**.

---

## Что намеренно не входило в этап

- **Backtest**, графики, тренды, скользящие окна по истории.
- Ограничение размера **`check_history.json`** и ротация **`app.log`** (заложена возможность расширения).
- Отдельный слой «аналитики» поверх истории — только хранение и просмотр.

## Итог этапа

Каждая успешная **`run_check`** (в том числе в составе **`check --all`**) сохраняет снимок метрик в **`data/check_history.json`**; **`range history`** позволяет смотреть последние записи по монете или глобально; **`data/logs/app.log`** фиксирует вызовы команд и типичные сбои рынка и оценки. **`show`** показывает последний статус и время проверки без вывода всей истории в консоль.
