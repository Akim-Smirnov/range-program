# Этап 2: каркас CLI и структура проекта

Документ фиксирует, **что реально сделано** на этом этапе в репозитории Range Program, в соответствии с общим планом (`plan.md`) и целью этапа: **фундамент без бизнес-логики расчёта диапазонов**.

---

## Цель этапа

Получить **минимально жизнеспособный каркас**: структура каталогов, разделение слоёв, рабочий CLI с базовыми командами управления списком монет и простым локальным хранилищем. Это база для следующих этапов (рынок, расчёты, `check`, `recalc` и т.д. из плана).

---

## Что сделано

### 1. Структура проекта

Используется раскладка **src-layout** (пакет в `src/`), чтобы установка через `pip install -e .` и импорты были предсказуемыми.

- **`src/range_program/`** — основной пакет приложения.
- **`src/range_program/cli.py`** — интерфейс командной строки (Typer).
- **`src/range_program/models/`** — модели данных (пока только `Coin`).
- **`src/range_program/repositories/`** — доступ к данным (JSON).
- **`src/range_program/services/`** — тонкий слой над репозиторием (место для будущей оркестрации без смешивания с CLI).
- **`data/`** — каталог для файлов данных (в репозитории есть `data/.gitkeep`; `coins.json` создаётся при первой записи).
- **`tests/`** — автотесты репозитория (pytest).

В корне: **`pyproject.toml`** (зависимости, точка входа `range`, метаданные пакета), **`README.md`** с краткой установкой и запуском.

### 2. Инструменты

- **Python** (в `pyproject.toml` указано `requires-python >= 3.10`).
- **Typer** — декларативный CLI, команды `add`, `remove`, `list`, `show`.
- Сознательно **не** подключались pandas, HTTP-клиенты, API бирж и прочие тяжёлые зависимости.

### 3. Модель данных

Сущность **`Coin`**:

- `symbol` — тикер в **верхнем регистре** (нормализация при создании и в репозитории).
- `created_at` — момент добавления (**UTC**, ISO 8601 при сериализации).

Расширенные поля из `plan.md` (источник данных, таймфрейм, активный диапазон и т.д.) **на этом этапе не вводились** — только минимум для списка монет.

### 4. Хранилище

- Файл **`data/coins.json`**.
- Содержимое: JSON-массив объектов `{ "symbol": "...", "created_at": "..." }`.
- Репозиторий сам создаёт каталог `data/` и файл при необходимости.
- **Дубликаты по символу не допускаются** при добавлении.

### 5. Команды CLI

| Команда | Назначение на этапе 2 |
|--------|------------------------|
| `range add <symbol>` | Добавить монету; при повторе — сообщение без дублирования записи. |
| `range remove <symbol>` | Удалить монету; при отсутствии — ненулевой код выхода. |
| `range list` | Вывести все монеты (символ и дата создания). |
| `range show <symbol>` | Показать одну монету: `symbol` и `created_at`. |

Точка входа: консольный скрипт **`range`** (см. `[project.scripts]` в `pyproject.toml`). Дополнительно: **`python -m range_program`** через `__main__.py`.

### 6. Разделение слоёв

Соблюдено разделение, заявленное в задаче этапа:

- **CLI** (`cli.py`) — разбор аргументов, вывод в консоль, коды выхода; не содержит работы с файлами напрямую.
- **Service** (`services/coin_service.py`) — вызовы репозитория; готов к расширению без разрастания CLI.
- **Repository** (`repositories/coin_repository.py`) — чтение/запись JSON, список монет, добавление/удаление.
- **Model** (`models/coin.py`) — тип данных монеты.

### 7. Тесты

В **`tests/test_coin_repository.py`** проверяются сценарии: пустое хранилище, добавление, отказ при дубликате, удаление, корректность JSON после записи.

---

## Связь с `plan.md`

В **`plan.md`** для полной версии описаны расширенные сценарии: `range check`, `range recalc`, активный диапазон, цены, статусы. **На этапе 2 это сознательно не реализовывалось.**

Реализована только **минимальная подмножина раздела «Управление монетами»** (`add`, `remove`, `list`, `show`) в упрощённом виде: без режимов, параметров и первичного расчёта — ровно то, что нужно для каркаса.

---

## Что намеренно не входило в этап

- Расчёт рекомендуемых диапазонов, волатильность, аналитика.
- Рыночные данные и интеграции с биржами.
- Команды `check`, `recalc`, `set-active` и остальные из плана полной версии.
- Расширенная модель монеты и журналы проверок.

## Как запускать (напоминание)

```bash
pip install -e .
range --help
range add BTC
range list
range show BTC
```

Или:

```bash
python -m range_program list
```
