# Этап 13: интерактивное меню в терминале (menu mode)

Документ фиксирует, **что реализовано** в репозитории на этом этапе. Цель: дать второй способ работы с программой — **пошаговое меню со стрелками и Enter** — **поверх** уже существующих сервисов и CLI, без дублирования бизнес-логики и без отказа от командного режима.

Связанные документы: [`etap-12.md`](etap-12.md) (рынок и fallback; меню использует тот же `MarketDataService`); предыдущие этапы с CLI-командами — [`etap-2.md`](etap-2.md) и далее по плану.

**Принцип архитектуры:** слой меню (**навигация, ввод, вывод**) только **вызывает** `CoinService`, `MarketDataService`, `RecalcService`, `CheckService`, `CheckHistoryRepository`, `run_backtest`, `compare_modes` / `best_mode_result` и общие функции отчёта — те же контракты, что и Typer CLI. Логика расчёта диапазонов, проверок и хранения **не переносится** в `menu.py`.

---

## Проблема до этапа

- Вся работа шла через **отдельные команды** (`range add`, `range recalc`, `range check`, …): удобно для скриптов и автоматизации, но для интерактивного сценария пользователь каждый раз набирает полную команду с флагами.
- Не было **единого сценария** «открыл программу → выбрал раздел → ответил на вопросы» без знания синтаксиса CLI.

---

## Цель этапа

1. Команда **`range menu`** (и alias **`range ui`**) запускает **интерактивный режим**.
2. **Главное меню** с разделами: Coins, Market data, Range analysis, Checks, History, Backtest, Optimize, Выход.
3. Внутри разделов — действия, соответствующие уже существующим командам (список монет, add с пошаговыми полями, recalc, check, history, backtest, optimize / suggest и т.д.).
4. **Выбор монеты из списка** из хранилища (если список не пуст), вместо обязательного ручного ввода тикера строкой.
5. После выполнения действия — **не выходить из программы**: предложить «Назад в раздел», «Главное меню» или «Выход».
6. **Ошибки** (монета не найдена, нет рынка, нет active range, API недоступно, неверное число) — **краткое сообщение**, без traceback в нормальном сценарии.
7. Для деструктивных операций — **подтверждение** (удаление монеты, сброс capital, сброс active range).
8. Для опциональных полей при добавлении монеты — **пустой Enter = пропуск** (capital, exchange, quote).
9. **Зависимость:** библиотека **`questionary`** (лёгкий prompt/menu, не full-screen ncurses).

## Что сделано

### 1. Модуль меню

Файл: **`src/range_program/menu.py`**.

- **`MenuDeps`** — контейнер зависимостей (те же сервисы, что создаются в `cli.py` для команд).
- **`run_interactive_menu(deps)`** — главный цикл: главное меню → вызов секций → выход по «Выход» или цепочке «Дальше».
- Секции: **`_coins_section`**, **`_market_section`**, **`_range_section`**, **`_checks_section`**, **`_history_section`**, **`_backtest_section`**, **`_optimize_section`**.
- **`prompt_next_step`** — единый шаг «Дальше» после действия (назад / главное меню / выход).
- **`_pick_coin_symbol`** — выбор монеты из отсортированного списка или «Отмена»; при пустом хранилище — понятное сообщение.
- Чистые хелперы парсинга ввода (для тестов и единообразия): **`parse_optional_float`**, **`parse_optional_str`**, **`parse_int_with_default`**.
- Обработка ошибок в действиях через **`_safe_call`**: перехват **`ValidationError`**, **`MarketDataError`**, **`RangeEngineError`**, **`ValueError`** с выводом короткого текста.

### 2. Общий вывод для CLI и меню

Файл: **`src/range_program/display_helpers.py`**.

- Вынесены из CLI функции печати:
  - **`print_grid_setups_block`** — блок вариантов сетки после recalc;
  - **`print_mode_comparison_table`** — таблица сравнения режимов (optimize / suggest).
- **`src/range_program/cli.py`** импортирует их вместо локальных **`_print_*`**, чтобы не дублировать разметку вывода между командным режимом и меню.

### 3. Интеграция в Typer CLI

Файл: **`src/range_program/cli.py`**.

- **`_run_interactive_menu()`** — собирает **`MenuDeps`** из модульных **`_service`**, **`_market`**, **`_recalc`**, **`_check`**, **`_history_repo`** и вызывает **`run_interactive_menu`**.
- Команды:
  - **`range menu`** — интерактивное меню;
  - **`range ui`** — то же поведение (alias).

Все **существующие команды** (`add`, `recalc`, `check`, …) **сохранены** без изменения семантики.

### 4. Зависимости

Файл: **`pyproject.toml`**.

- Добавлено: **`questionary>=2.0.0`**.

## Соответствие разделов меню и уже существующей логики

| Раздел меню        | Использование сервисов / модулей |
|--------------------|----------------------------------|
| Coins              | `CoinService` (list, get, add, remove, capital, active range, clear), `MarketDataService.resolve_market` + `update_coin` для resolve-market |
| Market data        | `get_price_quote`, `get_ohlcv`, resolve-market как выше |
| Range analysis     | `RecalcService.recalc`, чтение `recommended_range` / grid с монеты |
| Checks             | `CheckService.run_check`, `run_check_all`; последняя проверка — поля `Coin.last_check` |
| History            | `CheckHistoryRepository.get_last_n`, `get_global_last_n` + **`print_history_entries`** |
| Backtest           | **`run_backtest`** с передачей `market=` |
| Optimize           | **`compare_modes`**, **`best_mode_result`**; режим suggest дублирует смысл CLI **`suggest`** (таблица + текстовые рекомендации без автосохранения) |

---

## Что намеренно не входило в этап

- Full-screen TUI (ncurses, textual и т.п.), мышь, веб-интерфейс.
- Перенос бизнес-логики из сервисов в слой меню.
- Полный e2e-тест всего дерева меню с реальным вводом в терминале (достаточно точечных тестов и ручной проверки).
- Отдельный пакет «фронтенда» вне репозитория.

## Связь с целями проекта

Интерактивный режим снижает порог входа для сценария «посмотреть монеты → пересчитать диапазон → проверить статус → посмотреть историю» без запоминания флагов CLI, при этом **command mode** остаётся основным для автоматизации и документации.

---

## Итог этапа

Программа поддерживает **два режима**: классический **Typer CLI** и новый **`range menu` / `range ui`** с пошаговыми промптами на **questionary**, переиспользующий те же сервисы и общие блоки вывода (**`display_helpers`**). Бизнес-логика остаётся в сервисах и репозиториях; меню — только оркестрация ввода-вывода.
