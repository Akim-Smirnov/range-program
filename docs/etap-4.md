# Этап 4: подключение рыночных данных

Документ фиксирует, **что реально сделано** на этом этапе в репозитории Range Program. Цель этапа: **слой получения рыночных данных** (текущая цена и исторические свечи) через единую библиотеку **ccxt**, с **Binance** и парой **BASE/USDT** по умолчанию — **без расчёта рекомендуемого диапазона, без ATR/волатильности/статусов и без команд `check` / `recalc`**.

Связанные документы: общий план — [`plan.md`](plan.md); этап 3 (модель и JSON) — [`etap-3.md`](etap-3.md); этап 2 — [`etap-2.md`](etap-2.md).

---

## Цель этапа

Дать программе возможность:

1. запрашивать **текущую цену** по символу монеты;
2. запрашивать **исторические свечи OHLCV** с параметрами таймфрейма и лимита;
3. **встроить это в CLI** и в команду **`show`**, не смешивая HTTP/ccxt с логикой CLI в одном файле и не превращая ошибки сети в длинный traceback для пользователя.

Формат хранения монет в **`data/coins.json`** на этом этапе **не менялся** — рынок подставляется при запросе (см. ниже).

## Что сделано

### 1. Зависимость

В **`pyproject.toml`** в зависимости проекта добавлено:

- **`ccxt>=4.0.0`** — унифицированный доступ к биржам (в коде используется класс биржи по идентификатору, по умолчанию `binance`).

---

### 2. Конфигурация источника

Файл: **`src/range_program/config.py`**.

- **`DEFAULT_EXCHANGE_ID`** — `"binance"` (spot через ccxt).
- **`DEFAULT_QUOTE_ASSET`** — `"USDT"`: торговая пара собирается как **`BASE/USDT`** (например `BTC` → **`BTC/USDT`**).

---

### 3. Модель свечи `Candle`

Файл: **`src/range_program/models/candle.py`**.

Неизменяемый dataclass с полями:

- **`timestamp`** (`datetime`, UTC при разборе из биржи);
- **`open`**, **`high`**, **`low`**, **`close`**, **`volume`** (float).

Экспорт из **`src/range_program/models/__init__.py`**.

---

### 4. Сервис `MarketDataService`

Файл: **`src/range_program/services/market_data.py`**.

Назначение: **вся работа с ccxt** сосредоточена здесь; репозиторий JSON и CLI напрямую ccxt не вызывают.

Сущности:

- **`MarketDataError`** — исключение с **человекочитаемым** сообщением для вывода в CLI (сеть, неверный символ, ответ биржи, пустые данные и т.д.).
- **`PriceQuote`** (`NamedTuple`) — **`price`** и **`as_of`** (момент времени для цены: timestamp тикера с биржи в UTC или время запроса).

Публичные возможности:

- **`pair_for_symbol(symbol)`** — строка рынка вида **`BASE/QUOTE`** (база нормализуется, при строке с **`/`** берётся часть до слэша).
- **`get_price_quote(symbol)`** — цена (**`last`**, при отсутствии — **`close`**) и время.
- **`get_current_price(symbol) -> float`** — упрощённый контракт «только цена» (реализован через **`get_price_quote`**).
- **`get_ohlcv(symbol, timeframe, limit) -> list[Candle]`** — **`fetch_ohlcv`** ccxt, разбор строк в **`Candle`**.

Технические детали:

- экземпляр биржи создаётся лениво, **`enableRateLimit: True`**;
- ошибки ccxt маппятся в **`MarketDataError`** (в т.ч. сеть, несуществующая пара, общие ошибки биржи);
- пустой список свечей или неразборчивый ответ обрабатываются явным сообщением.

Экспорт из **`src/range_program/services/__init__.py`**: **`MarketDataService`**, **`MarketDataError`**, **`PriceQuote`**.

---

### 5. Обновление CLI

Файл: **`src/range_program/cli.py`**.

Принцип: обращение к рынку только через **`_market = MarketDataService()`**, без прямых вызовов ccxt в обработчиках команд (кроме косвенно через сервис).

Изменения и новые команды:

| Команда / поведение | Описание |
|---------------------|----------|
| **`range show SYMBOL`** | После полей монеты и **`active_range`** выполняется попытка **`get_price_quote`**. При успехе выводятся **`last_price`** и **`price_as_of`**. При **`MarketDataError`** — жёлтое сообщение **без падения** процесса с traceback. |
| **`range price SYMBOL`** | Только котировка: символ, пара, цена, время. Ошибка рынка → красное сообщение и код выхода **1**. |
| **`range candles SYMBOL --limit N`** | Монета должна **существовать в локальном хранилище**; **`timeframe`** берётся из **`coin.timeframe`**; вывод таблично (время UTC и OHLCV). **`--limit`** по умолчанию **10**, значения меньше **1** отклоняются. |

Команды управления списком монет (**`add`**, **`remove`**, **`list`**, **`set-active`**, **`clear-active`**) смыслово не заменялись; к ним добавился рыночный слой только там, где описано выше.

---

### 6. Обработка ошибок (ожидания этапа)

- **Монета не в хранилище** — для **`candles`** (и для **`show`**, как раньше): понятное сообщение, для отсутствующей при **`show`** — выход с кодом **1** по-прежнему.
- **Биржа / сеть / нет пары / пустой ответ** — переводится в **`MarketDataError`** и краткий текст для пользователя; для **`price`** и **`candles`** при фатальной ошибке — выход **1** без длинного traceback.

### 7. Тесты

Файл: **`tests/test_market_data.py`**.

Проверяются без сети (моки): сборка пары, **`get_current_price`**, разбор **`get_ohlcv`**, реакция на пустой список свечей.

Полный прогон: **`python -m pytest tests`**.

## Что намеренно не входило в этап

- Расчёт **рекомендуемого диапазона**, **ATR**, **волатильность**, статусы **`OK` / `WARNING`** и т.п.
- Команды **`range check`**, **`range recalc`** и любые «оценщики» диапазона.
- Смена формата **`coins.json`** под котировки (цены в файле не кэшируются на этом этапе).
- Отдельная СУБД — по-прежнему только локальный JSON для монет.

Появился **изолированный слой рыночных данных** (**`MarketDataService`** + модель **`Candle`** + конфиг), **зависимость ccxt**, **обновлённый CLI** с **`price`**, **`candles`** и рыночным блоком в **`show`**.
