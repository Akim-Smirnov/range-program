# Инструкция по использованию Range Program

Это **консольная программа**: команды вводятся в терминале, данные хранятся локально в `data/`, при обращении к бирже запрашиваются только **рыночные котировки и свечи** (чтение), без торговли по API.

С **этапа 13** доступны **два способа работы**: привычный **командный режим** (`range add`, `range recalc`, …) и **интерактивное меню** (`range menu`). Оба используют одни и те же сервисы и файлы в `data/`.

## 1. Установка и запуск

Из каталога проекта (где лежит `pyproject.toml`):

```bash
pip install -e .
```

Проверка:

```bash
range --help
```

Точка входа в `pyproject.toml` — команда **`range`**. Эквивалент:

```bash
python -m range_program --help
```

**Требования:** Python 3.10+ (см. `pyproject.toml`), стабильный **интернет** для команд с биржей. Основные зависимости: **Typer**, **ccxt**, **questionary** (меню). Отдельный `requirements.txt` или `.env` для базовой работы **не обязателен**.

## 2. Два способа работы

### Командный режим (CLI)

Вы вводите **одну команду целиком**: подкоманда, аргументы (часто тикер), опции с `--` или `-`. Это удобно для скриптов, планировщиков, копирования примеров из документации и быстрых одноразовых действий.

**Общий вид:**

```text
range <подкоманда> [аргументы] [опции]
```

Примеры:

```bash
range add BTC
range recalc ETH --help
range check --all
```

Справка по любой подкоманде:

```bash
range add --help
range check --help
```

### Интерактивное меню

Запуск:

```bash
range menu
```

То же самое (псевдоним):

```bash
range ui
```

Откроется **текстовое меню** (стрелки вверх/вниз, Enter): разделы вроде Coins, Market data, Range analysis, Checks и т.д. Внутри — пошаговые вопросы (выбор монеты из списка, лимиты, подтверждения удаления). **Логика та же**, что у CLI: вызываются те же сервисы, данные пишутся в те же файлы. Подробности и ограничения — в **[`etap-13.md`](etap-13.md)**.

В разделе **Range analysis** есть два варианта пересчёта:
- **recalc** — пересчитать по сохранённым настройкам монеты и сохранить результат;
- **Recalc с параметрами (анализ/override)** — “примерить” `timeframe/lookback/методы` **без записи**, а затем при желании **сохранить** рассчитанный результат и параметры в монету.

**Когда что использовать**

| Ситуация | Удобнее |
|----------|---------|
| Скрипт, cron, повторяемый сценарий | CLI |
| Освоение программы без запоминания флагов | `range menu` |
| Одна короткая операция из памяти | CLI |

## 3. Сводка команд `range` (командный режим)

Все подкоманды можно посмотреть через `range --help`. Ниже — краткая карта.

| Подкоманда | Назначение |
|------------|------------|
| **`menu`**, **`ui`** | Запуск интерактивного меню (см. §2). |
| **`add`** | Добавить монету: тикер и опции `--mode`, `--timeframe`, `--lookback`, `--center-method`, `--width-method`, `--capital`, `--exchange`, `--quote`. |
| **`remove`** | Удалить монету из списка. |
| **`list`** | Список всех монет. |
| **`show`** | Карточка монеты (параметры, диапазоны, кэш рынка, последняя цена). |
| **`price`** | Текущая цена (для монеты из списка — с fallback по биржам). |
| **`candles`** | Свечи OHLCV; таймфрейм из монеты; `--limit` / `-n`. |
| **`resolve-market`** | Найти и сохранить в монете рабочую биржу и пару. |
| **`recalc`** | Пересчитать рекомендуемый диапазон и сохранить. |
| **`set-capital`** | Задать капитал (для вариантов сетки при `recalc`). |
| **`clear-capital`** | Сбросить капитал. |
| **`set-active`** | Задать активный диапазон: `--low`, `--high`, опционально `--comment`. |
| **`clear-active`** | Сбросить активный диапазон. |
| **`check`** | Проверка одной монеты или **`--all`** / **`-a`** — все монеты. |
| **`history`** | История проверок: по символу или **`--all`**, опционально **`--limit`**. |
| **`backtest`** | Упрощённый backtest на истории; обязательно **`--days`** / **`-d`**. |
| **`optimize`** | Сравнение режимов ширины диапазона на истории; **`--days`**. |
| **`suggest`** | То же сравнение + текстовая подсказка сменить `mode` вручную; **`--days`** по умолчанию как lookback. |

Детали аргументов и значений по умолчанию — в справке Typer, например:

```bash
range add --help
range backtest --help
```

## 4. Общая логика работы

1. **Добавить монету** (`range add` или меню **Coins → Add coin**) — режим ширины диапазона, таймфрейм, при желании биржу/котировку и капитал.
2. **Пересчитать рекомендуемый диапазон** (`range recalc` / меню **Range analysis**) — загрузка рынка, расчёт в `RangeEngine`, сохранение в `coins.json`; при заданном капитале — блок **вариантов сетки**. Для подбора параметров можно использовать **Recalc с параметрами (анализ/override)** в меню.
3. **Задать активный диапазон** (`range set-active` / меню) — ваша реальная сетка. В меню можно указать **комментарий** к active range (опционально), а при **сбросе** active range — тоже оставить комментарий (он попадёт в лог).
4. **Проверять** (`range check` / `check --all` или меню **Checks**) — согласованность активного диапазона с ценой и рекомендацией; записи попадают в **историю**.
5. По необходимости — **backtest** и **сравнение режимов** (`backtest`, `optimize`, `suggest` или соответствующие пункты меню).

Все данные монет и журналы — в папке **`data/`** (см. раздел 11).

## 5. Управление списком монет

| Действие | Команда |
|----------|---------|
| Добавить | `range add BTC` |
| С параметрами | `range add ETH --mode balanced --timeframe 4h --lookback 60 --center-method ema --width-method atr` |
| С капиталом (для сетки после `recalc`) | `range add SOL --capital 1000` |
| Предпочтительная биржа и котировка | `range add HYPE --exchange bybit --quote USDT` (коротко: `-e`, `-q`; подробнее — [`etap-12.md`](etap-12.md)) |
| Список | `range list` (в т.ч. колонка **capital**) |
| Карточка | `range show BTC` |
| Удалить | `range remove BTC` |

**`--mode` (ширина диапазона в движке):** `conservative` | `balanced` | `aggressive` — влияют на множители ATR и ограничения ширины ([`etap-5.md`](etap-5.md)).

**`--timeframe`:** строка в стиле ccxt, например `4h`, `1d`, `1h`.

**Дефолты для новой монеты** задаются в коде (`defaults.py`): например `balanced`, `4h`, lookback 60 дней, `center_method=ema`, `width_method=atr`, если не переопределить.

**`center_method`** (центр рекомендуемого диапазона в `RangeEngine`): `price` | `ema` | `sma` | `median` | `midpoint` | `donchian` (период окна для исторических методов в MVP — 20 свечей, см. `range_engine.py`). По умолчанию **`ema`**.

**`width_method`** (полуширина коридора вокруг центра): `atr` (ATR × множитель по `mode`) | `std` (σ по close × множитель по `mode`) | `donchian` | `historical_range` (для двух последних в MVP — полуширина `(max high − min low) / 2` по окну 20 свечей). По умолчанию **`atr`**.

## 6. Капитал и варианты сетки

Поверх **рекомендуемого диапазона** можно получить три **профиля геометрической сетки** (число уровней, шаг %, размер ордера в котируемом активе), если задан **капитал** по монете ([`etap-11.md`](etap-11.md)).

**Важно:** у **`coin.mode`** (conservative / balanced / aggressive) и у **профиля сетки** (`GridConfig`) те же имена, но **разный смысл**: первое — **ширина коридора диапазона**, второе — **плотность сетки** (агрессивная сетка = больше уровней, мельче шаг).

| Команда | Назначение |
|---------|------------|
| `range set-capital BTC 5000` | Задать капитал в **дефолтной котировке** из конфига (обычно USDT) |
| `range clear-capital BTC` | Убрать капитал (после `recalc` блок сетки не считается) |

Капитал можно указать при **`range add … --capital`**. После **`range recalc`** при ненулевом капитале выводится блок **Grid setups**; без капитала — жёлтая подсказка про `set-capital`. В **`show`** — строка capital и **Recommended grid setups**, если они есть в сохранённом `recommended_range`.

Ограничения: **нет** учёта комиссий, минимального лота биржи и автоторговли — только справочные цифры.

## 7. Рынок: цена, свечи, поиск пары

| Команда | Назначение |
|---------|------------|
| `range price BTC` | Текущая цена. **Монета в списке** — используется **`MarketDataService`** с fallback по биржам и котировкам, кэш **`resolved_*`** в монете ([`etap-12.md`](etap-12.md)). **Монеты нет в списке** — одна дефолтная биржа и котировка из `config.py` (часто Binance + USDT). |
| `range candles BTC` | Свечи; таймфрейм из карточки монеты. `--limit` / `-n` — сколько свечей. Монета должна быть в списке. |
| `range resolve-market HYPE` | Явно найти рабочую биржу и пару, вывести результат и **сохранить** `resolved_exchange`, `resolved_symbol_pair`, `resolved_at` в монете. |

Порядок бирж (MVP): **binance → bybit → okx → kucoin**; котировки: **USDT → USDC → USD** (с учётом `--exchange` / `--quote` у монеты). В **`show`** видны предпочтения и закэшированная пара.

## 8. Рекомендуемый и активный диапазон

| Команда | Назначение |
|---------|------------|
| `range recalc BTC` | Загрузить OHLCV и цену, посчитать **recommended_range**, сохранить; обновить кэш рынка. При капитале — **grid_configs** внутри рекомендации. После основного блока: **таблица сравнения всех `center_method`** при текущем `width_method` монеты; затем **таблица сравнения всех `width_method`** (atr, std, donchian, historical_range) при текущем `center_method`. В `data/` сохраняется только выбранная пара методов; в таблицах `*` отмечает сохранённую строку. |
| `range set-active BTC --low 60000 --high 72000` | **Активный** диапазон (ваша сетка). Опционально `--comment "..."`. |
| `range clear-active BTC` | Сбросить активный диапазон. В меню при сбросе можно добавить комментарий (логируется). |

**`check`** для монеты требует **активного** диапазона. Если **нет `recommended_range`**, при первом **`check`** выполняется автоматический **`recalc`** ([`etap-6.md`](etap-6.md)).

Алгоритм диапазона (MVP): центр **EMA(20)** или **price**, ширина через **ATR(14)** и множители по режиму, затем clamp по % — детали в [`etap-5.md`](etap-5.md).

## 9. Проверка состояния: статусы и история

### Команды

| Команда | Назначение |
|---------|------------|
| `range check BTC` | Оценка активного диапазона; обновление **`last_check`**, запись в **`check_history.json`**. |
| `range check --all` или `-a` | Все монеты: **таблица** + **Summary**; сбой по одной монете даёт строку **ERROR**, прогон не прерывается ([`etap-7.md`](etap-7.md)). |
| `range history BTC` | Последние записи по монете (по умолчанию 10). `--limit` / `-n` |
| `range history --all` | Последние записи по всем символам (по умолчанию 20; можно `--limit`). |

Нужно указать **либо** символ, **либо** `--all` (для `check` и `history`).

### Статусы (MVP, приоритет сверху вниз)

См. [`etap-6.md`](etap-6.md): **`OUT_OF_RANGE`** → **`REPOSITION`** (сильное расхождение центров) → **`STALE`** (большое отклонение цены от центра активного) → **`WARNING`** (близко к границе) → **`OK`**.

## 10. Анализ на истории

| Команда | Назначение |
|---------|------------|
| `range backtest BTC --days 30` | Упрощённый сценарий «жизни» диапазона: фиксация диапазона после прогрева, шаг по **`close`**, без учёта high/low внутри свечи ([`etap-9.md`](etap-9.md)). Обязателен `--days` / `-d`. Лимит запроса свечей — **не более 1000**; при нехватке окна — уменьшите `--days` или смените таймфрейм монеты. |
| `range optimize BTC --days 60` | Три режима **ширины диапазона** на одной истории; таблица и **Best mode** по **score** ([`etap-10.md`](etap-10.md)). |
| `range suggest BTC` | То же сравнение; `--days` по умолчанию как lookback (60). Подсказка **вручную** сменить `mode` и выполнить **`recalc`** — **автосохранения режима нет**. |

**Score** (эвристика): учитываются «дни жизни», доли OK/STALE, максимальное отклонение; WARNING в формулу score не входит ([`etap-10.md`](etap-10.md)).

Backtest и optimize — **не** полноценный симулятор прибыли, комиссий и проскальзывания нет.

## 11. Где что хранится

| Путь | Содержимое |
|------|------------|
| `data/coins.json` | Монеты: параметры, **active_range**, **recommended_range** (в т.ч. **grid_configs**), **last_check**, кэш рынка (**resolved_***), **capital**, preferred **exchange** / **quote_asset** и др. |
| `data/check_history.json` | Журнал проверок (каждая успешная **`run_check`**, в том числе из **`check --all`**) ([`etap-8.md`](etap-8.md)). |
| `data/logs/app.log` | Файловый лог: вызовы команд, предупреждения рынка и оценки. Ротация размера файла в коде **не** реализована. |

Резервное копирование: достаточно папки **`data/`**.

## 13. Типичные сценарии

**Первый запуск и проверка диапазона:**

```bash
range add BTC
range recalc BTC
range set-active BTC --low ... --high ...
range check BTC
```

**То же через меню:** `range menu` → **Coins** → добавить монету → **Range analysis** → recalc → (опционально “сохранить recommended_range как active range” с комментарием) → **Coins** / команды CLI для `set-active` → **Checks**.

**Подбор параметров через меню:** `range menu` → **Range analysis** → **Recalc с параметрами (анализ/override)** → посмотреть результат и диагностику → при необходимости сохранить → дальше как обычно (active range и checks).

**С капиталом и вариантами сетки:**

```bash
range add ETH --capital 2000
range recalc ETH
range set-active ETH --low ... --high ...
range check ETH
```

**Ежедневный обзор портфеля:**

```bash
range check --all
```

**Сравнение режимов ширины диапазона на истории:**

```bash
range optimize BTC --days 60
range suggest BTC
```

Дальше при смене режима правьте **`mode`** в `data/coins.json` или пересоздайте монету с `range add … --mode …`, затем снова **`range recalc`**.
